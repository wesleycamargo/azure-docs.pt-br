# <a name="tutorial-train-a-classification-model-with-automated-machine-learning-in-azure-machine-learning-service"></a>Tutorial: Treinar um modelo de classificação com aprendizado de máquina automatizado no serviço do Azure Machine Learning

Neste tutorial, você aprenderá a gerar um modelo de machine learning usando ML (aprendizado de máquina) automatizado.  O serviço do Azure Machine Learning pode executar o pré-processamento de dados, a seleção de algoritmo e a seleção de hiperparâmetro de maneira automatizada para você. O modelo final pode então ser implantado seguindo o fluxo de trabalho no tutorial [Implantar um modelo](tutorial-deploy-models-with-aml.md).

![diagrama de fluxo](./media/tutorial-auto-train-models/flow2.png)

Semelhante ao [tutorial para treinar modelos](tutorial-train-models-with-aml.md), este tutorial classifica imagens manuscritas de dígitos (0-9) do conjunto de dados [MNIST](http://yann.lecun.com/exdb/mnist/). Porém, neste momento, você não especifica um algoritmo nem ajusta os hiperparâmetros. A técnica de ML automatizado itera em muitas combinações de algoritmos e hiperparâmetros até encontrar o melhor modelo com base em seu critério.

Você aprenderá a:

> [!div class="checklist"]
> * Configurar seu ambiente de desenvolvimento
> * Acesse e examine os dados
> * Treinar usando um classificador automatizado em seu computador local
> * Explorar os resultados
> * Examinar os resultados de treinamento
> * Registrar o melhor modelo

Se você não tiver uma assinatura do Azure, crie uma [conta gratuita](https://azure.microsoft.com/free/?WT.mc_id=A261C142F) antes de começar.

## <a name="set-up-your-development-environment"></a>Configurar seu ambiente de desenvolvimento

Toda a configuração para o seu trabalho de desenvolvimento pode ser realizada em um Python Notebook.  A instalação inclui:

* Importar pacotes do Python
* Configure um workspace para habilitar a comunicação entre seu computador local e recursos remotos
* Crie um diretório para armazenar os scripts de treinamento

### <a name="import-packages"></a>Importe os pacotes
Importe os pacotes do Python de que você precisa nesta sessão.



Depois que você tiver um objeto de workspace, especifique um nome para o experimento e crie e registre um diretório local com o workspace. O histórico de todas as execuções é registrado no experimento especificado.

## <a name="explore-data"></a>Explorar dados

O tutorial do treinamento inicial usava uma versão de alta resolução do conjunto de dados MNIST (28 x 28 pixels).  Como o treinamento automatizado de ML requer muitas iterações, este tutorial usa uma versão de resolução menor das imagens (8 x 8 pixels) para demonstrar os conceitos enquanto acelera o tempo necessário para cada iteração.  

```python
from sklearn import datasets

digits = datasets.load_digits()

# Exclude the first 100 rows from training so that they can be used for test.
X_train = digits.data[100:,:]
y_train = digits.target[100:]
```

### <a name="display-some-sample-images"></a>Exibir algumas imagens de exemplo

Carregue os dados em um matrizes `numpy`. Em seguida, use `matplotlib` para plotar 30 imagens aleatórias do conjunto de dados com seus rótulos acima delas.

## <a name="train-a-model"></a>Treinar um modelo

Para treinar o modelo automaticamente, primeiro defina parâmetros de configuração para o experimento e, em seguida, execute o experimento.


### <a name="define-settings"></a>Definir configurações

Defina as configurações do experimento e as configurações do modelo.

|Propriedade| Valor neste tutorial |DESCRIÇÃO|
|----|----|---|
|**primary_metric**|AUC ponderada | Métrica que você deseja otimizar.|
|**max_time_sec**|12.000|Limite de tempo em segundos para cada iteração|
|**iterações**|20|Número de iterações. Em cada iteração, o modelo treina com os dados com um pipeline específico|
|**n_cross_validations**|3|Número de divisões de validação cruzada|
|**preprocess**|Falso| *True/False* habilita o experimento a executar o pré-processamento na entrada.  Pré-processamento lida com os *dados ausentes* e executa alguma *extração de recurso* comum|
|**exit_score**|0,9985|Valor *duplo* indicando o destino para *primary_metric*. Depois que o destino for ultrapassado, a execução será encerrada|
|**blacklist_algos**|['kNN','LinearSVM']|*Matriz* de *cadeias de caracteres* indicando os algoritmos a ignorar.
|

```python
from azureml.train.automl import AutoMLConfig

##Local compute 
Automl_config = AutoMLConfig(task = 'classification',
                             primary_metric = 'AUC_weighted',
                             max_time_sec = 12000,
                             iterations = 20,
                             n_cross_validations = 3,
                             preprocess = False,
                             exit_score = 0.9985,
                             blacklist_algos = ['kNN','LinearSVM'],
                             X = X_train,
                             y = y_train,
                             path=project_folder)
```

### <a name="run-the-experiment"></a>Execute o experimento

Inicie o experimento a ser executado localmente. Definir o destino de computação como o local e defina a saída como true para exibir o progresso do experimento.


```python
from azureml.core.experiment import Experiment
experiment=Experiment(ws, experiment_name)
local_run = experiment.submit(Automl_config, show_output=True)
```

## <a name="register-the-best-model"></a>Registrar o melhor modelo 

Use o objeto `local_run` para obter o melhor modelo e registrá-lo no workspace. 

```python
# find the run with the highest accuracy value.
best_run, fitted_model = local_run.get_output()

# register model in workspace
description = 'Automated Machine Learning Model'
tags = None
local_run.register_model(description=description, tags=tags)
local_run.model_id # Use this id to deploy the model as a web service in Azure
```

## <a name="test-the-best-model"></a>Teste o melhor modelo

Use o modelo para prever alguns dígitos aleatórios.  Exiba o valor e a imagem previstos.  A fonte vermelha e a imagem inversa (branco sobre preto) são usadas para realçar as amostras classificadas incorretamente.

Como a precisão do modelo é alta, talvez seja necessário executar o código a seguir algumas vezes, antes de poder ver uma amostra classificada incorretamente.

```python
# find 30 random samples from test set
n = 30
X_test = digits.data[:100, :]
y_test = digits.target[:100]
sample_indices = np.random.permutation(X_test.shape[0])[0:n]
test_samples = X_test[sample_indices]


# predict using the  model
result = fitted_model.predict(test_samples)

# compare actual value vs. the predicted values:
i = 0
plt.figure(figsize = (20, 1))

for s in sample_indices:
    plt.subplot(1, n, i + 1)
    plt.axhline('')
    plt.axvline('')
    
    # use different color for misclassified sample
    font_color = 'red' if y_test[s] != result[i] else 'black'
    clr_map = plt.cm.gray if y_test[s] != result[i] else plt.cm.Greys
    
    plt.text(x = 2, y = -2, s = result[i], fontsize = 18, color = font_color)
    plt.imshow(X_test[s].reshape(8, 8), cmap = clr_map)
    
    i = i + 1
plt.show()
```
